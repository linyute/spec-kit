# Spec Kit

*更快地建立高品質軟體。*

**一項旨在讓組織專注於產品情境，而不是在 Spec-Driven Development 的幫助下編寫無差異程式碼的努力。**

## 什麼是 Spec-Driven Development？

Spec-Driven Development **顛覆了**傳統軟體開發的模式。數十年來，程式碼一直是王道 — 規格只是我們在開始編碼的「真正工作」時建立和丟棄的鷹架。Spec-Driven Development 改變了這一點：**規格變得可執行**，直接產生可運作的實作，而不僅僅是引導它們。

## 核心理念

Spec-Driven Development 是一個結構化的流程，強調：

- **意圖驅動開發**，其中規格在「如何做」之前定義「做什麼」
- **豐富的規格建立**，使用護欄和組織原則
- **多步驟細化**，而不是從提示中一次性產生程式碼
- **高度依賴**先進的 AI 模型能力來解釋規格

## 開發階段

| 階段 | 焦點 | 主要活動 |
|-------|-------|----------------|
| **從 0 到 1 開發** (「綠地」) | 從頭開始產生 | <ul><li>從高層次需求開始</li><li>產生規格</li><li>規劃實作步驟</li><li>建立可投入生產的應用程式</li></ul> |
| **創意探索** | 平行實作 | <ul><li>探索多樣化的解決方案</li><li>支援多種技術堆疊和架構</li><li>實驗使用者體驗模式</li></ul> |
| **迭代增強** (「棕地」) | 棕地現代化 | <ul><li>迭代新增功能</li><li>現代化舊有系統</li><li>調整流程</li></ul> |

## 實驗目標

我們的研究和實驗重點是：

### 技術獨立性

- 使用多樣化的技術堆疊建立應用程式
- 驗證 Spec-Driven Development 是一個不依賴特定技術、程式語言或框架的流程的假設

### 企業限制

- 展示關鍵任務應用程式開發
- 納入組織限制 (雲端供應商、技術堆疊、工程實踐)
- 支援企業設計系統和合規性要求

### 以使用者為中心的開發

- 為不同的使用者群體和偏好建立應用程式
- 支援各種開發方法 (從 vibe-coding 到 AI 原生開發)

### 創意和迭代流程

- 驗證平行實作探索的概念
- 提供強大的迭代功能開發工作流程
- 擴展流程以處理升級和現代化任務
